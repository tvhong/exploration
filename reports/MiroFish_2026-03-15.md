# MiroFish

> A universal swarm intelligence prediction engine that builds parallel digital worlds with autonomous multi-agent simulations to rehearse complex scenarios and forecast outcomes.

## What is it?

MiroFish is an AI-powered prediction and simulation platform that tackles the problem of understanding complex system behavior and predicting future trajectories in uncertain environments. The core insight is that by extracting seed information from real-world data (news, policy documents, financial reports, or even fiction), you can construct a high-fidelity digital mirror populated with thousands of autonomous agents, each with distinct personalities, long-term memory, and behavioral logic.

The motivation stems from decision-making challenges at scale: policymakers need to test policy decisions at zero risk, public relations teams want to forecast opinion dynamics, novelists wish to explore narrative branches, and analysts need to understand how information cascades through social networks. Traditional prediction methods are often linear, static, and fail to capture the emergent complexity of human behavior. MiroFish reimagines prediction as a simulation sandbox where users can inject variables, observe cascading effects across an agent population, and generate data-driven narratives about "what if" scenarios.

The platform is developed by the MiroFish team with strategic support from Shanda Group and uses OASIS (a social media simulation framework by CAMEL-AI) as its core simulation engine.

## Ecosystem fit & innovations

MiroFish sits at the intersection of several domains: multi-agent systems, knowledge graph reasoning (via Zep), large language models, and social network simulation. Its key innovations include:

1. **GraphRAG-based Context Building**: Uses Zep Cloud's knowledge graph capabilities to extract entities and relationships from raw text, building a structured ontology that serves as the foundation for agent personalities and world state.

2. **Dual-Platform Social Simulation**: Runs parallel simulations on Twitter and Reddit-like platforms simultaneously, allowing realistic social media dynamics to unfold. Agents can create posts, like, repost, follow, quote, and interact naturally.

3. **Autonomous Agent Persona Generation**: Rather than scripting agent behavior, MiroFish uses LLMs to generate rich, multi-dimensional personas (personality, background, beliefs, motivations) that drive authentic interactions. Each agent maintains long-term memory via Zep.

4. **Interactive Simulation Environment**: Post-simulation, users can chat directly with any agent in the digital world or interact with a ReportAgent (powered by ReACT patterns) that uses a rich toolset to analyze, search, and interpret simulation results.

5. **Project-centric Architecture**: Implements a persistent project context system that maintains state across multiple steps (graph building → environment setup → simulation → reporting → interaction), allowing users to revisit and refine their digital worlds.

Unlike traditional forecasting tools, MiroFish doesn't predict via regression or time-series models—it predicts through **behavioral emergence**: simulating how individual decisions compound into collective phenomena.

## How it works & data flow

### Workflow Overview

MiroFish implements a 5-stage linear workflow:

```
Stage 1: Graph Building
  Input: Raw text (reports, articles, novels)
    ↓
  Ontology Generation (LLM analyzes text → entity/relationship types)
    ↓
  Graph Construction (Zep builds knowledge graph from chunked text)
    ↓
  Output: Zep graph_id + metadata

Stage 2: Environment Setup
  Input: Zep graph_id + simulation requirements (natural language)
    ↓
  Entity Reading & Filtering (Extract only social-media-compatible entities)
    ↓
  Agent Persona Generation (LLM enriches entities → detailed OASIS profiles)
    ↓
  Simulation Configuration (Set parameters: max rounds, agent list, actions)
    ↓
  Output: OASIS simulation config ready to execute

Stage 3: Simulation Execution
  Input: OASIS config + simulation parameters
    ↓
  Multi-Round Social Simulation (Twitter + Reddit platforms in parallel)
    ↓
  Action Logging (Capture each agent action per round with timestamp/result)
    ↓
  Memory Updates (Zep graph memory enriched with simulation outcomes)
    ↓
  Output: simulation_id + action history + memory updates

Stage 4: Report Generation
  Input: simulation_id + simulation requirement
    ↓
  ReportAgent Planning (Outline sections to address user's prediction needs)
    ↓
  Multi-Round Generation (ReACT: LLM writes → uses tools → reflects → refines)
    ↓
  Tool Usage (Search graph, forge insights, generate panoramic views, interview agents)
    ↓
  Output: Structured prediction report with interpretations

Stage 5: Interactive Exploration
  Input: simulation_id + user queries
    ↓
  Direct Agent Interview (Query individual agents about their actions/beliefs)
    ↓
  ReportAgent Dialogue (Ask agent questions; it searches, analyzes, responds)
    ↓
  Output: Conversational insights into simulation behavior
```

### Key Technical Details

**Graph Construction**: Text is split into overlapping chunks (default 500 tokens with 50-token overlap), sent in batches to Zep Cloud, which uses its own NLP to extract entities and relationships. The resulting graph is stored in Zep's backend and identified by `graph_id`.

**Agent Personality Generation**: Each entity node in the graph becomes an OASIS agent. The OASIS Profile Generator enriches basic entity info (name, type, summary) with LLM-generated details: persona descriptions, background stories, personality traits (MBTI-style), motivations, and communication style. This ensures agents don't follow rigid scripts but behave autonomously.

**Simulation Mechanics**: OASIS simulates agents on Twitter and Reddit simultaneously. In each round, agents perceive the current state, decide which action to take (CREATE_POST, LIKE, REPOST, FOLLOW, etc.), and execute it. Results are logged per agent per round. The simulation continues for a user-specified number of rounds (default 10).

**Memory & Context**: Zep maintains a persistent memory for each agent, storing prior interactions, beliefs, and outcomes. During simulation, the memory system is updated with agent actions, making the graph evolve as the simulation progresses.

**Report Generation**: The ReportAgent uses a ReACT (Reasoning, Acting, Thinking) loop: it reasons about what sections to generate, calls tools (search graph for relevant entities, forge insights, get panoramic views of narratives, interview specific agents), processes results, reflects on quality, and refines. This iterative approach ensures nuanced, multi-perspective reports.
